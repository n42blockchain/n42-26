# Devlog 130 — Gov5 Status session compatibility

Date: 2026-07-22

## Outcome

The read-only Rust interop observer now completes Gov5's mandatory
`/rpc/status/1/ssz_snappy` admission handshake. Gov5 therefore keeps the
TCP/Noise/Yamux session alive instead of disconnecting an otherwise compatible
observer after its ten-second inbound-handshake deadline.

This is enabled only on the explicit Gov5 TCP observer swarm. Native Rust
validator swarms keep their previous transport and protocol surface.

## Wire boundary

The codec implements Gov5's exact 72-byte Status SSZ shape:

- two dynamic offsets (`8`, `40`);
- the genesis `H256` limb byte order used by Gov5;
- a 256-bit current height, accepted only when it fits `u64`;
- uvarint uncompressed-length prefix;
- Snappy framed stream rather than raw Snappy blocks;
- response status byte before the length-prefixed payload.

The reader terminates on a complete Snappy frame yielding the declared Status
length rather than treating connection EOF as the message boundary. The same
bidirectional libp2p stream remains available for the response. Frame count, chunk size, decoded
size, response code, and SSZ layout are bounded and fail closed.

A fixed fixture generated by `github.com/golang/snappy` is decoded in Rust. A
temporary Gov5 source-level test also decoded the Rust-produced framed bytes
with `SszNetworkEncoder.DecodeWithMaxLength`; the temporary test file was then
removed and the Gov5 worktree returned clean.

## Scheduling boundary

Gov5 requires an inbound peer to initiate Status within ten seconds. The
observer queues the request on `ConnectionEstablished` and flushes it at the
start of the next network-loop iteration, before continuously busy replay and
block-fetch queues. This avoids both opening a substream from inside the
connection callback and starving the admission handshake during catch-up.

## Live evidence and next gate

A correctly rebuilt `n42-node-bin` observer connected to a temporary Gov5
validator. The Status exchange completed in about two milliseconds, returned
Gov5 height 1601, and the connection remained established beyond the previous
ten-second cutoff with zero disconnects.

The stable session isolates the next catch-up boundary: starting from the
authenticated replay-v2/QMDB block-49 checkpoint while Gov5 was above block
1600 exceeded the 512-entry live window, so the Rust execution head correctly
remained at block 49. Far-distance joining therefore still needs an
authenticated checkpoint/finalized-range continuation before validator
participation is enabled.

All live work used temporary data. Existing seven-node, replay-history, and
performance datasets were not modified.
