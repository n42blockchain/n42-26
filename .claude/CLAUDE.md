# N42-26 项目规范

## 开发日志要求

每次完成阶段性工作（Phase 完成、重大功能实现、重要 bug 修复）后，必须将以下内容增量追加到项目根目录的 `开发日志.md` 文件中：

1. **设计决策**：为什么选择某种方案，考虑过哪些替代方案
2. **实施细节**：关键代码结构、API 使用方式、模块依赖关系
3. **遇到的问题及解决方案**：编译错误、API 不兼容、性能问题等
4. **阶段完成状态**：计划项清单和完成情况
5. **后续计划**：下一阶段的预览和待办

日志采用增量追加模式，每个阶段/功能作为新章节添加到文件末尾，保留完整历史。

## 项目上下文

- 自定义区块链系统：执行端基于 reth v1.11.0（本地 path 依赖 `../reth`），共识端采用 HotStuff-2 变体
- 分发节点（IDC）负责出块、共识投票、存储状态
- 手机进行并行验证，不在共识关键路径上
- 初始规模：100-500 分发节点，每节点连接约 10,000 手机
- 性能目标：8 秒 slot
